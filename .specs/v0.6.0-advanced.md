# v0.6.0 — Advanced Features

## Goal

Add release channel support and export/import for sharing package lists between machines.

---

## Feature 1: Release Channels

**What:** Install pre-release versions via channel tags (`--channel nightly`, `--channel beta`, `--channel stable`).

**Spec:**
- New flag: `gitclaw install user/repo --channel <name>`
- Channels map to GitHub release tag patterns:
  - `stable` — latest non-prerelease (current default behavior)
  - `beta` — latest release with `-beta` or `-rc` suffix in tag
  - `nightly` — latest release with `-nightly` or `-dev` suffix, or the newest prerelease
- If no `--channel` flag: current behavior (latest stable release)
- If `--channel stable`: explicitly requests stable (same as default)
- If `--channel beta`/`nightly`: fetch all releases, filter by tag pattern, pick newest match
- `gitclaw search user/repo --channel <name>` filters releases by channel
- Channel config can be overridden in `.gitclaw.toml`:
  ```toml
  [channels]
  nightly = ["*-nightly", "*-dev"]
  beta = ["*-beta", "*-rc"]
  stable = ["!*-*"]   # exclude prerelease patterns
  ```
- Default channel patterns are built-in; config extends/overrides them

**Checkpoint:** `gitclaw install BurntSushi/ripgrep --channel nightly` installs the newest nightly/dev build; `--channel beta` installs the newest beta/rc.

---

## Feature 2: Export / Import

**What:** Share installed package lists between machines via TOML files.

**Spec:**
- `gitclaw export` — writes installed packages to stdout as TOML
- `gitclaw export -o deps.toml` — writes to file instead of stdout
- `gitclaw import deps.toml` — installs all packages listed in the file
- Export format:
  ```toml
  [[package]]
  owner = "BurntSushi"
  repo = "ripgrep"
  version = "13.0.0"     # pinned version

  [[package]]
  owner = "sharkdp"
  repo = "fd"
  version = "8.7.0"
  ```
- Import reads the file and runs `install` for each entry (respects cache)
- `--channel` from export is preserved in the TOML (optional field)
- Import skips already-installed packages unless `--force` is set
- Export output is deterministic: sorted by owner then repo

**Checkpoint:** `gitclaw export -o deps.toml` on machine A, `gitclaw import deps.toml` on machine B installs the same packages.
