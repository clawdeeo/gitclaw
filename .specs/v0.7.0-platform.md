# v0.7.0 — Platform Integration

## Goal

Persist the install channel in the registry so `gitclaw update` respects it.

---

## Feature: Update with Channel

**What:** Store the channel a package was installed from, and use it when updating.

**Spec:**
- Add optional `channel` field to `InstalledPackage` with `#[serde(default)]`
- When a package is installed with `--channel`, store the channel string in the registry
- `gitclaw update` reads the stored channel and uses it when checking for updates
- If no channel stored, current behavior (latest stable)
- Backward compatible: existing registries without `channel` field load normally

**Checkpoint:** Install with `--channel nightly`, then `gitclaw update` fetches the newest nightly.
